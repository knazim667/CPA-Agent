## Task 1: Real-time UI polling

**Files:**
- Modify: `ui/app.js:936`

**Context:** `fetchStatus()` is called exactly once in `DOMContentLoaded`. There is no recurring poll, no WebSocket. The dashboard, model badge, and conversation list are frozen until the user manually reloads.

- [ ] **Step 1: Add a 5-second poll after initial load**

  ```javascript

  ```

- [ ] **Step 2: Refresh status after every mutating API call**

  Each of these functions makes a POST and receives `data.status` — they already call `updateStatus(data.status)` on success. That means the poll covers background changes. No extra changes needed for the chat, transaction form, document approval, or provider/mode switches — they all update status inline.

  However `fetchLedger()` is called only on tab switch and pagination clicks. Add a call to `fetchLedger(currentLedgerPage)` inside `fetchStatus` when the ledger tab is active, so ledger data refreshes automatically.

  Find the `fetchStatus` function (around line 856–865):

  ```javascript
  function fetchStatus() {
    fetch('/api/status')
      .then(function (r) { return r.json(); })
      .then(function (data) {
        updateStatus(data);
        // Refresh ledger if that tab is active
        var ledgerBtn = document.querySelector('.tab-btn[data-tab="ledger"]');
        if (ledgerBtn && ledgerBtn.classList.contains('active')) {
          fetchLedger(currentLedgerPage || 1);
        }
      })
      .catch(function (err) {
        if (bootWarning) { bootWarning.classList.remove('hidden'); }
        console.error('fetchStatus error:', err);
      });
  }
  ```

  Start the server: `source .venv/bin/activate && uvicorn web_app:app --reload`

  Open `http://localhost:8000`. Open DevTools → Network tab. Confirm a `GET /api/status` request fires every ~5 seconds.

**Context:** `list_businesses()` returns `[{"key": "nazam_llc", "business_name": "Nazam LLC"}, ...]`.       opt.textContent = biz.name || biz.key;
  ```javascript
      if (biz.key === status.active_business_key) { opt.selected = true; }

  ```

  Replace with:

  ```javascript
  if (businessSelect && status.businesses) {
    businessSelect.textContent = '';
  ```

## Task 3: Duplicate transaction detection

**Files:**
- Modify: `skills/google_sheets_manager.py` — add `find_duplicate_row()`
- Modify: `main.py` — call duplicate check before `append_ledger_row` in `record_structured_transaction()`
- Modify: `tests/test_main_fixes.py` — add duplicate detection test

**Context:** There is zero deduplication logic. If the same transaction is submitted twice (via chat and manual form, or by uploading a receipt twice), both rows are appended without any warning.

**Strategy:** Before appending, read the last 100 rows of the ledger and check for an exact match on `(date, amount, type)`. If found, block the append and return a warning message. The user can override explicitly.

- [ ] **Step 1: Add `find_duplicate_row()` to `GoogleSheetsManager`**

  In `skills/google_sheets_manager.py`, after the `append_ledger_row` method (after line 179), add:

  ```python
  def find_duplicate_row(
      self,
      spreadsheet_id: str,
      date: str,
      amount: str | float,
      entry_type: str,
      lookback_rows: int = 100,
  ) -> dict[str, Any] | None:
      """Return the first row that matches date+amount+type, or None."""
      rows = self.read_range(spreadsheet_id, f"Ledger!A2:G{1 + lookback_rows}")
      target_amount = str(amount).strip()
      target_date = str(date).strip()
      target_type = str(entry_type).strip().lower()
      for row in rows:
          if len(row) < 5:
              continue
          row_date = str(row[0]).strip()
          row_amount = str(row[3]).strip()
          row_type = str(row[4]).strip().lower()
          if row_date == target_date and row_amount == target_amount and row_type == target_type:
              return {
                  "date": row[0],
                  "description": row[1] if len(row) > 1 else "",
                  "amount": row[3],
                  "type": row[4],
              }
      return None
  ```

## Task 4: Graceful OCR fallback

**Files:**
- Modify: `skills/document_processor.py` — guard pytesseract, return helpful error

**Context:** `pytesseract` requires the system-level Tesseract binary to be installed. If it's not installed (which is likely on a fresh macOS setup), any image upload silently crashes. The user sees a generic server error instead of a helpful message.

- [ ] **Step 1: Read the current document_processor.py**

  Open `skills/document_processor.py` and find the `_extract_image_text` or OCR-related method. Note the exact import location of `pytesseract`.

- [ ] **Step 2: Guard the import and provide a fallback**

  At the top of `skills/document_processor.py`, replace any top-level `import pytesseract` with a guarded import:

  ```python
  try:
      import pytesseract
      _OCR_AVAILABLE = True
  except ImportError:
      _OCR_AVAILABLE = False
  ```

  Then in the method that calls `pytesseract.image_to_string(...)`, wrap it:

  ```python
  if not _OCR_AVAILABLE:
      return "[OCR unavailable — install Tesseract: brew install tesseract]"
  ```

- [ ] **Step 3: Check if Tesseract binary is installed and report clearly**

  In the same method, after the `_OCR_AVAILABLE` check, add a runtime check:

  ```python
  import shutil
  if not shutil.which("tesseract"):
      return "[OCR unavailable — Tesseract binary not found. Run: brew install tesseract]"
  ```

  Upload a `.txt` or `.pdf` file via the Documents tab. Confirm it processes correctly. Upload a `.jpg` or `.png`. Confirm it returns the OCR-unavailable message instead of crashing.
